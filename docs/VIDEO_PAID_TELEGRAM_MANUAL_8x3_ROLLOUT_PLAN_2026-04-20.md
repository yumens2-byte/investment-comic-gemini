# 비디오 트랙 수동 검증(유료 텔레그램) 8초×3컷, 3회 실행 설계 (2026-04-20)

## 1) 목표
완전 자동 배포 전, **유료 텔레그램 채널 운영 흐름에서 Gemini 기반 영상 생성 품질/안정성/운영성**을 먼저 검증한다.

- 실행 단위: **1회 = 24초 (8초 × 3컷)**
- 검증 횟수: **총 3회(수동)**
- 검증 채널: **유료 텔레그램 전용**

---

## 2) 왜 수동 3회가 필요한가
1. 프롬프트가 실제 장세(시장 컨텍스트)에 맞게 스토리 다양성을 만드는지 확인
2. 캐릭터 일관성(히어로/빌런 고정)과 금지 규칙(브랜드/텍스트/저작권)을 점검
3. 자동화 전, 운영 오퍼레이션(작성→생성→검수→게시)의 병목을 확인

---

## 3) 수동 3회 운영 시나리오

### Run #1 — Baseline (표준 장세)
- 목적: 기본 프롬프트 품질 확인
- 입력: 당일 시장 데이터 + 기본 시나리오
- 합격 기준: 컷 연결 자연스러움, 캐릭터 고정, 24초 완주율

### Run #2 — Volatility Stress (고변동 장세)
- 목적: HIGH 리스크에서도 메시지 과장/붕괴 없이 전달되는지 확인
- 입력: VIX/WTI 고위험 구간 데이터
- 합격 기준: 과도한 노이즈/아티팩트 없음, 위기감은 상승

### Run #3 — Calm Regime (저변동 장세)
- 목적: LOW 리스크에서도 단조롭지 않게 스토리 유지되는지 확인
- 입력: NORMAL/INTEL 유형 데이터
- 합격 기준: NO_BATTLE/ONE_VS_ONE 전환이 자연스럽고 반복감 낮음

---

## 4) 입력 데이터 규격 (수동 작성자 체크리스트)
- 날짜: `YYYY-MM-DD`
- 시나리오: `ONE_VS_ONE | NO_BATTLE | ALLIANCE`
- 리스크: `LOW | MEDIUM | HIGH`
- 이벤트 타입: `BATTLE | SHOCK | AFTERMATH | INTEL | NORMAL | FLASHBACK | TACTICAL`
- 히어로/빌런 ID
- 핵심 시장 포인트 3개 (예: VIX, WTI, SPY)

---

## 5) 수동 생성 플로우 (Paid Telegram)
1. 템플릿 프롬프트에 시장값/캐릭터/시나리오 주입
2. 8초×3컷 생성 요청(컷별 prompt 분리 권장)
3. 컷 연결(순서/톤/색감) 확인
4. 내부 검수(금칙어, 캐릭터 붕괴, 텍스트 노출 여부)
5. 유료 텔레그램 업로드 + 운영 로그 기록

---

## 6) QA 기준 (Go/No-Go)
### 필수 통과
- 24초(8×3) 구조 유지
- 캐릭터 정체성 유지(성별/연령/의상/종족 불변)
- 금지 요소 미노출(브랜드/실존 인물/저작권 캐릭터/과도한 텍스트)
- 스토리 전달 가능(도입→위협→대치)

### 계량 지표
- 생성 실패율: 3회 기준 10% 이하
- 재생성 횟수: 회차당 평균 1.5회 이하
- 수동 소요시간: 회차당 30분 이하

---

## 7) 자동화 전환 조건
아래 3개 모두 충족 시 자동화 전환:
1. 수동 3회 모두 필수 통과
2. 계량 지표 충족
3. 유료 텔레그램 운영자 피드백에서 품질 수용

---

## 8) 리스크 및 대응
- 리스크: 캐릭터 형태 붕괴
  - 대응: CHARACTER LOCK 블록 강화 + 컷별 동일 키워드 고정
- 리스크: 과도한 텍스트/로고 생성
  - 대응: Negative Prompt 고정 문구 재사용
- 리스크: 컷 간 톤 불일치
  - 대응: Cut1 end-frame hint를 Cut2 첫 프레임 제약으로 연결
