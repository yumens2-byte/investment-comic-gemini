# 24초 뉴스형 히어로 vs 빌런 예고편 상세 설계

## 1) 프로젝트 목표
- **목표 결과물**: 24초 분량의 숏폼 영상(9:16 기준, 1080x1920), 뉴스 속보 포맷 기반의 예고편.
- **콘셉트**: 실제 뉴스 속보처럼 시작해, 마지막에 히어로-빌런 대결을 암시하는 티저로 전환.
- **핵심 감정선**: 긴급함(0~8s) → 불안 고조(8~16s) → 기대/전율(16~24s).

## 2) 타깃/톤앤매너
- **타깃**: 금융·시사·히어로 장르를 함께 소비하는 20~40대 모바일 시청자.
- **톤**:
  - 앵커 파트: 객관적·긴박한 뉴스 톤.
  - 예고 파트: 영화 트레일러 톤(저음 내레이션 + 임팩트 자막).
- **비주얼 스타일**: 블루/레드 대비, 글리치 전환, 브레이킹 뉴스 lower-third.

## 3) 화면 포맷 및 기술 사양
- **해상도**: 1080x1920 (9:16)
- **프레임레이트**: 30fps
- **총 길이**: 24.0s
- **오디오**:
  - 마스터 -14 LUFS
  - 피크 -1dB 이하
  - 뉴스 SFX + 트레일러 히트 사운드 레이어링

## 4) 타임라인 상세 설계 (24초)

### 0.0s ~ 3.0s (오프닝 속보)
- **화면**: 뉴스 오프닝 그래픽 + "BREAKING NEWS" 배너.
- **자막**: "긴급 속보 | 도심 에너지 이상 징후 포착"
- **오디오**: 뉴스 알림음(2회) + 빠른 드럼 펄스.
- **목적**: 시청자 주목 즉시 확보.

### 3.0s ~ 7.0s (앵커 브리핑)
- **화면**: 앵커 샷(중앙 구도), 우측 상단 LIVE 배지.
- **앵커 대사(예시)**:  
  "방금 들어온 소식입니다. 정체불명의 빌런 신호가 도시 전역에서 감지됐습니다."
- **그래픽**: 하단 자막 + 우측에 도시 지도 인포그래픽.
- **목적**: 사건의 신뢰감 부여(뉴스 문법 고수).

### 7.0s ~ 11.0s (빌런 예고)
- **화면**: 빌런 실루엣, 붉은 경보 라인, 글리치 확대.
- **자막**: "코드명: BLACKOUT / 위험 등급 MAX"
- **효과**: 0.5초 간격 점멸 컷 3회.
- **사운드**: 저주파 럼블 + 금속성 히트.
- **목적**: 위협의 크기 시각화.

### 11.0s ~ 15.0s (히어로 등장 암시)
- **화면**: 역광 속 히어로 실루엣, 망토/장비 디테일 스냅 컷.
- **자막**: "미확인 대응자 출현"
- **내레이션(예시)**:  
  "그러나, 그를 막을 단 한 명의 이름이 다시 호출되고 있습니다."
- **목적**: 대결 구도 완성.

### 15.0s ~ 20.0s (대치 몽타주)
- **화면**: 히어로/빌런 교차 편집(0.7~1.0초 컷), 화면 중앙 분할 대치.
- **자막 키워드**: "충돌", "선택", "카운트다운"
- **사운드**: 심장박동 + 서브드롭 + 상승 스트링.
- **목적**: 클라이맥스 전 긴장 최대치.

### 20.0s ~ 24.0s (티저 클로징)
- **화면**: 암전 직전 충돌 프레임 정지 → 타이틀 카드.
- **타이틀**: "HERO vs VILLAIN : PRELUDE"
- **CTA**: "매주 일·화 공개 | FOLLOW FOR ALERT"
- **오디오**: 임팩트 1회 후 잔향 페이드아웃.
- **목적**: 다음 콘텐츠 기대감 및 팔로우 유도.

## 5) 내레이션/자막 카피 초안

### 앵커/내레이션 결합 6문장 구조
1. "긴급 속보입니다. 도심 전역에 비정상 에너지 파형이 발생했습니다."
2. "당국은 코드명 BLACKOUT의 재등장을 공식 확인했습니다."
3. "시민 대피령이 검토되는 가운데, 한 인물의 신호가 포착됐습니다."
4. "그의 귀환은 희망일까요, 더 큰 전쟁의 시작일까요."
5. "정면충돌까지 남은 시간, 단 24초."
6. "히어로와 빌런의 서막, 곧 공개됩니다."

## 6) 제작 리소스 설계
- **필수 소스**
  - 앵커 배경(뉴스 스튜디오 이미지/영상) 1종
  - 빌런 실루엣 2~3컷
  - 히어로 실루엣/장비 디테일 2~3컷
  - 도시 야경/경보 HUD 오버레이
- **그래픽 패키지**
  - Lower-third 2종(속보형, 인터뷰형)
  - LIVE 배지 1종
  - 타이틀 카드 1종
- **사운드**
  - 뉴스 알림 SFX, 저주파 럼블, 히트, 심장박동 루프, 트레일러 브람.

## 7) 편집 룰(실무 적용)
- 컷 길이는 **평균 0.8~1.2초**, 정보 전달 컷(앵커)은 2초 이상 유지.
- 자막은 한 줄 14자 내외, 최대 2줄.
- 글리치/셰이크 효과는 빌런 등장 컷에만 집중 사용(과용 금지).
- 색보정:
  - 뉴스 파트: 채도 -10, 대비 +5
  - 티저 파트: 대비 +20, 블루/레드 스플릿 톤

## 10) 일/화 발행 운영 설계

### 발행 원칙
- **정기 발행 요일**: 매주 **일요일, 화요일** 고정 업로드
- **권장 발행 시간(한국 기준)**: 오후 7시~9시
- **시리즈 운영 템포**:
  - 일요일: 스토리 전개/정보 공개형
  - 화요일: 충돌 암시/클리프행어형

### 콘텐츠 캘린더(4주 예시)
- **1주차 일**: Ep.0 예고 (히어로 vs 빌런 서막)
- **1주차 화**: Ep.1 히어로 기원
- **2주차 일**: Ep.2 빌런 목적
- **2주차 화**: Ep.3 첫 충돌 직전
- **3주차 일**: Ep.4 전황 브리핑(뉴스 특집 형식)
- **3주차 화**: Ep.5 양측 전략 공개
- **4주차 일**: Ep.6 시민/도시 피해 스냅샷
- **4주차 화**: Ep.7 라운드1 결전 티저

### 일/화 포맷 차별화 가이드
- **일요일 포맷(정보형 뉴스)**
  - 앵커 비중 40% 이상
  - 인포그래픽/지도/수치 자막 적극 활용
  - 다음 화요일 예고를 2초 포함
- **화요일 포맷(예고형 트레일러)**
  - 몽타주 컷 비중 60% 이상
  - 대사 최소화 + 임팩트 카피 중심
  - 다음 주 일요일 본편 유도 CTA 삽입

### 일/화 CTA 문구 표준안
- 일요일 종료 카피: "다음 전개는 화요일 속보에서 공개됩니다"
- 화요일 종료 카피: "다음 본편은 일요일 20:00 공개"

## 11) 확장안 (후속 시리즈)
- **Ep.0 예고(현재)**: 등장 암시
- **Ep.1**: 히어로 기원
- **Ep.2**: 빌런 목적
- **Ep.3**: 첫 충돌 하이라이트

2. 일요일용 자막 패키지(정보형)와 화요일용 자막 패키지(예고형) 2트랙 제작

## 14) 파일럿 테스트 및 반영 프로세스

### 파일럿 테스트 범위
- **대상 버전**: Ep.0 24초 티저(일/화 공통 포맷 기준)
- **테스트 채널**: 내부 시사 링크 + 비공개 업로드
- **테스트 인원**: 최소 5명(기획 1, 편집 1, 마케팅 1, 일반 시청자 2)

### 테스트 체크 항목
1. **첫 2초 훅 체감**: 시선을 붙잡는지 5점 척도 평가
2. **메시지 명확성**: 히어로/빌런 대결 구도가 즉시 이해되는지 확인
3. **자막 가독성**: 모바일 6인치 화면 기준 판독 가능 여부
4. **오디오 전달력**: 대사/효과음 밸런스, 피로도, 볼륨 이슈 점검
5. **CTA 전환성**: "다음 공개 요일(일/화)" 인지가 되는지 확인

### 파일럿 운영 일정(개발완료 후)
- **D+0**: 개발완료본 동결(파일명 `pilot_v1`) 및 내부 공유
- **D+1**: 파일럿 시청/설문 회수(정량+정성)
- **D+2**: 이슈 분류(P0/P1/P2) 및 수정 적용
- **D+3**: 반영본 `release_candidate` 확정 후 예약 업로드

### 반영 우선순위 규칙
- **P0(즉시 수정)**: 오탈자, 요일 오류, 심각한 음량/싱크 문제
- **P1(출시 전 수정 권장)**: 자막 가독성 저하, 컷 템포 불균형
- **P2(다음 회차 반영)**: 연출 취향 차이, 미세한 컬러 톤 선호
